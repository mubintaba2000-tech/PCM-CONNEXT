# พิมพ์เขียวและแผนการพัฒนา (Roadmap)

เอกสารนี้จัดทำขึ้นเพื่อส่งต่อบริบทการทำงานให้ Agent เสมือนหรือทีมผู้พัฒนา (Antigravity/Gemini) ให้เข้าใจโครงสร้างและเป้าหมายต่อไปที่ต้องการพัฒนา

## การพัฒนาในระยะสั้น (Short-term Goals)
1. **ปรับปรุง UI/UX ของระบบจัดการข้อมูลโครงการ**: ทำให้ Responsive และใช้งานได้จริงทั้งบน Desktop และ Tablet 
2. **ระบบแผนงานและ Task Management**: 
   - เพิ่มระบบ Kanban Board / Gantt Chart ในหน้า Project Detail
   - ให้ Foreman/Engineer อัปเดต % ความคืบหน้าของงานย่อยได้ 
3. **ระบบจัดการไฟล์แนบ (File Management)**: 
   - รองรับการอัปโหลดไฟล์/รูปภาพหน้างานลงใน Firebase Storage (ยังไม่ได้ทำ)
   - เชื่อมโยงรูปภาพเข้ากับ Report ของแต่ละโฟร์แมน

## การพัฒนาในระยะกลาง (Medium-term Goals)
1. **ระบบออกเอกสาร (Document Generation)**:
   - แปลงข้อมูลใบเสนอราคา, อินวอยซ์, เอกสารรับเงิน ให้ออกมาในรูปแบบ PDF เพื่อให้พร้อมพิมพ์
2. **ระบบการแจ้งเตือน (Notifications)**:
   - แจ้งเตือนเมื่อมีการอนุมัติเอกสาร, มีการเปิดโพรเจกต์ใหม่, หรือมีงานใกล้ถึงกำหนด (Due Date)
3. **ฟังก์ชันขั้นสูงของ Admin/Owner**:
   - หน้า Dashboard สรุปผลกำไร/ขาดทุนของทุกโปรเจกต์เปรียบเทียบกัน
   - ระบบเช็คชื่อ / นำเข้าข้อมูลเวลาทำงานของพนักงานหน้างาน

## การพัฒนาในระยะยาว (Long-term Goals)
1. **ระบบสื่อสารกับลูกค้า (Client Portal)**:
   - ให้สิทธิ์ลูกค้า (`client`) ล็อกอินเข้ามาเพื่อดูความคืบหน้าของโครงการตัวเองเท่านั้น พร้อมรูปภาพรีวิวและแชทกับทีมงานได้
2. **เชื่อมต่อกับระบบอื่น (Third-party Integrations)**:
   - เชื่อมระบบบัญชี (เช่น FlowAccount, Xero, PEAK)
   - เชื่อมต่อการแจ้งเตือนเข้าสู่แอปพลิเคชัน LINE (LINE Notify / LINE Messaging API)

## ฟังก์ชันที่ยังขาดอยู่และต้องสร้างเพิ่มเติม (Backlog Modules)
- **Storage/File Upload API** - การจัดเก็บไฟล์ 
- **PDF Export** - ตัวช่วยแปลง Component UI เป็น PDF
- **Dynamic Security Rules สำหรับ Firestore** - ปัจจุบัน Rules ถูกปรับให้ทำงานแบบยืดหยุ่น(Demo)ไว้ ในการขึ้น Production จำเป็นต้องเขียน Rules ให้รัดกุมตาม Role ที่ใช้งานจริง
